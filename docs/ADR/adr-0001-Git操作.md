## コンテキスト

Gitの操作について、どのような方法で行うか

## 検討した選択肢

### 選択肢1: gitコマンド

- 概要: gitコマンドを実行してその標準出力や標準エラー出力をパースしつつ操作する
- メリット:
  - デバッグがしやすい
  - LLMの精度が高い
  - 細かいオプション指定ができる
  - 本家の実装なのでバグが少ない
- デメリット:
  - 標準出力などのパースが手間
  - 細かいオプションや操作がすぐには必要ない

### 選択肢2: [LibGit2Sharp](https://github.com/libgit2/libgit2sharp)

- 概要: ライブラリを通してC#のコードで操作する
- メリット:
  - 静的型付言語を介することでミスが少なくなる
  - 標準出力などのパースをせずに進捗などを取得できる
  - 直感的なコーディングが期待できる
- デメリット:
  - 細かい操作ができない可能性がある
  - 本家よりバグに遭遇する可能性が高くなる

## 決定

選択肢2: LibGit2Sharp を採用する。

- 理由:
  - まだ細かい操作は必要でない
  - gitコマンドは習熟が難しい
  - C#コードのまま直接操作を記載できることは見通しの良さとミスの減少につながると判断
